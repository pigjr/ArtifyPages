# TextToEpub v1.0 功能说明

## 🎯 项目概述
TextToEpub是一个基于本地大模型的文本处理工具，能够读取txt文件，分章节处理，生成摘要和配图，最终输出为epub格式。

## ✨ 核心功能

### 📝 文本处理
- **智能分章**: 支持多种章节标题格式识别
- **内容分割**: 按段落数量灵活分割章节
- **标题提取**: 自动提取章节标题用于显示

### 🤖 AI摘要生成
- **本地模型**: 使用OpenVINO优化的Qwen2.5-7B-Instruct模型
- **英文摘要**: 生成简洁的英文摘要，控制在70个token内
- **批量处理**: 支持多章节连续处理
- **进度保存**: 支持中断后继续处理

### 🎨 AI图像生成
- **LCM模型**: 使用LCM_Dreamshaper_v7快速生成图像
- **安全提示**: 自动添加安全关键词，避免不当内容
- **高质量输出**: 生成512x512高清图像
- **降级机制**: GPU失败自动降级到CPU

### 📊 统计监控
- **实时内存**: 显示当前内存使用百分比
- **时间统计**: 记录每章处理时间和平均时间
- **进度跟踪**: 显示处理进度和完成百分比
- **自动保护**: 内存超过85%自动保存退出

### 🔧 模型管理
- **预加载**: 程序启动时预加载所有模型
- **互斥锁**: 确保同时只有一个大模型工作
- **资源复用**: 模型加载一次，重复使用
- **内存优化**: 避免重复加载导致的内存泄漏

## 🛠️ 技术特性

### 模型配置
```python
# 文本模型
# OpenVINO/Qwen2.5-1.5B-Instruct-int4-ov
llm_model_id = "OpenVINO/Qwen2.5-7B-Instruct-int4-ov"
llm_model_path = "models/Qwen2.5-7B-Instruct-int4-ov"

# 图像模型  
image_model_id = "OpenVINO/LCM_Dreamshaper_v7-fp16-ov"
image_model_path = "models/LCM_Dreamshaper_v7-fp16-ov"
```

### 章节识别模式
- `第[一二三四五六七八九十百千万\d]+章[^\n]*\n`
- `Chapter\s+\d+[^\n]*\n`
- `第\d+节[^\n]*\n`
- `^\d+\.[^\n]*\n`
- `^[一二三四五六七八九十百千万]+、[^\n]*\n`

### 内存保护机制
- **监控频率**: 每章处理完成后检查
- **阈值设置**: 85%内存使用率
- **自动保存**: 触发阈值时自动保存进度
- **安全退出**: 提供用户友好的退出提示

## 📋 使用方法

### 基本用法
```bash
python3 main.py input.txt
```

### 高级选项
```bash
# 指定每章段落数
python3 main.py input.txt --paragraphs 30

# 从中断处继续
python3 main.py input.txt --resume
```

### 参数说明
- `txt_file`: 输入的txt文件路径（必需）
- `--paragraphs`: 每章段落数（默认20）
- `--resume`: 从上次中断处继续处理

## 📁 文件结构

### 输入文件
- `input.txt`: 原始文本文件
- `input.txt.progress`: 处理进度文件（自动生成）

### 输出文件
- `output/`: 输出目录
  - `chapter_1_image.png`: 章节配图
  - `chapter_2_image.png`: 章节配图
  - ...
  - `input_generated.epub`: 最终epub文件

## 🔧 系统要求

### 硬件要求
- **CPU**: 支持OpenVINO的x86_64处理器
- **内存**: 建议8GB以上（模型较大）
- **存储**: 至少10GB可用空间（模型+输出）
- **GPU**: 可选，Intel集成显卡或独立显卡
